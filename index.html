🔒 Lock

Los objetos Lock ofrecen el mismo propósito que synchronized —exclusión mutua— pero con mayor control y flexibilidad.

ReentrantLock tryLock() fairness deadlock prevention

Teoría

Teoría

¿Por qué usar Lock en lugar de synchronized?

Los bloques synchronized son sencillos pero limitados. Los objetos Lock añaden dos capacidades clave:

  • Intentar adquirir sin bloquearse: tryLock() devuelve true/false inmediatamente.
  • Política de equidad (fairness): garantiza que ningún thread espere indefinidamente.

Regla de oro: Siempre liberar el lock en un bloque finally para que se libere aunque haya excepciones.

Crear un Lock

Sin política de equidad
Lock lock = new ReentrantLock();
Con política de equidad (fair)
// El thread que más tiempo lleva esperando
// tiene prioridad para adquirirlo
Lock lock = new ReentrantLock(true);

Obtener y liberar

Patrón obligatorio con finally
lock.lock();           // bloquea hasta obtenerlo
try {
    // código protegido
} finally {
    lock.unlock();     // siempre se libera
}

Solo el thread que adquirió el lock puede liberarlo. Si otro intenta liberarlo se lanza IllegalMonitorStateException.

tryLock() — intentar sin bloquearse

tryLock() intenta obtener el lock y devuelve true si lo consigue o false si no. El thread no se bloquea en ningún caso.

Patrón básico con un lock
boolean lockAdquirido = false;
try {
    lockAdquirido = lock.tryLock();
    if (lockAdquirido) {
        // código protegido
    } else {
        // alternativa si no se pudo obtener
    }
} finally {
    if (lockAdquirido) lock.unlock();
}

Patrón con dos Locks — prevención de deadlock

Cuando necesitamos ambos locks a la vez, el patrón con tryLock() evita el deadlock clásico donde cada thread tiene un lock y espera el del otro.

Intentar adquirir los dos; si no, reintentar
while (true) {
    boolean lock1Adquirido = false;
    boolean lock2Adquirido = false;
    try {
        lock1Adquirido = lock1.tryLock();
        lock2Adquirido = lock2.tryLock();

        if (lock1Adquirido && lock2Adquirido) {
            // ambos adquiridos: hacer el trabajo
            break;
        }
        // si no los tenemos ambos, el finally los libera
        // y volvemos a intentarlo
    } finally {
        if (lock1Adquirido) lock1.unlock();
        if (lock2Adquirido) lock2.unlock();
    }
}

¿Por qué esto evita el deadlock? Si el thread A tiene lock1 y no consigue lock2, lo libera todo antes de reintentar. Así el thread B puede avanzar mientras A espera.

Resumen de métodos de ReentrantLock

MétodoDescripción
lock()Bloquea el thread hasta que el lock esté disponible.
unlock()Libera el lock. Solo el thread propietario puede llamarlo.
tryLock()Intenta adquirir. Devuelve true/false sin bloquear.
tryLock(tiempo, unidad)Intenta durante un tiempo máximo antes de devolver false.
isHeldByCurrentThread()Comprueba si el thread actual posee el lock.
getQueueLength()Número de threads esperando para adquirirlo.

📚 Copybook

Un gestor de libros permite copiar las páginas interiores no vacías de un libro al final de otro. El problema es que dos threads intentan copiar simultáneamente (A→B y B→A), produciendo resultados inconsistentes.

Objetivo: garantizar que, aunque se lancen ambas copias a la vez, cada operación se complete de forma íntegra antes de empezar la otra.

Código de partida

import java.util.ArrayList;
import java.util.List;
import java.util.concurrent.Executors;

public class Main {

    static class Libro {
        ArrayList<String> paginas;

        Libro(List<String> paginas) {
            this.paginas = new ArrayList<>(paginas);
        }

        void copiarPaginas(Libro libroOrigen) {
            for (String pagina : libroOrigen.obtenerPaginasInterioresNoVacias()) {
                int penultimaPosicion = Math.max(0, this.paginas.size() - 1);
                this.paginas.add(penultimaPosicion, pagina);
            }
        }

        List<String> obtenerPaginasInterioresNoVacias() {
            List<String> paginas = new ArrayList<>();
            for (String pagina : this.paginas.subList(1, this.paginas.size() - 1)) {
                if (!pagina.isEmpty()) paginas.add(pagina);
            }
            return paginas;
        }
    }

    public static void main(String[] args) {
        for (int i = 0; i < 100; i++) {
            final Libro libroA = new Libro(List.of("Portada-A","A1","A2","","A3","A4","Contraportada-A"));
            final Libro libroB = new Libro(List.of("Portada-B","B1","B2","B3","","B4","Contraportada-B"));

            try (var executor = Executors.newVirtualThreadPerTaskExecutor()) {
                executor.submit(() -> libroA.copiarPaginas(libroB));
                executor.submit(() -> libroB.copiarPaginas(libroA));
            }

            System.out.println("Libro A: " + libroA.paginas);
            System.out.println("Libro B: " + libroB.paginas);
            System.out.println();
        }
    }
}

Pista de diseño

  • 1
    Añade un campo Lock lock = new ReentrantLock(); a la clase Libro.
  • 2
    En copiarPaginas necesitas adquirir ambos locks: el del libro destino (this) y el del libro origen.
  • 3
    Usa el patrón while(true) { tryLock + tryLock; if(ambos) { …; break; } } para evitar deadlock.
import java.util.ArrayList;
import java.util.List;
import java.util.concurrent.Executors;
import java.util.concurrent.locks.Lock;
import java.util.concurrent.locks.ReentrantLock;

public class Main {

    static class Libro {
        ArrayList<String> paginas;
        final Lock lock = new ReentrantLock();

        Libro(List<String> paginas) {
            this.paginas = new ArrayList<>(paginas);
        }

        void copiarPaginas(Libro libroOrigen) {
            // Adquirir ambos locks sin riesgo de deadlock
            while (true) {
                boolean lockDestinoAdq = false;
                boolean lockOrigenAdq  = false;
                try {
                    lockDestinoAdq = this.lock.tryLock();
                    lockOrigenAdq  = libroOrigen.lock.tryLock();

                    if (lockDestinoAdq && lockOrigenAdq) {
                        for (String pagina : libroOrigen.obtenerPaginasInterioresNoVacias()) {
                            int pos = Math.max(0, this.paginas.size() - 1);
                            this.paginas.add(pos, pagina);
                        }
                        break;
                    }
                } finally {
                    if (lockDestinoAdq) this.lock.unlock();
                    if (lockOrigenAdq)  libroOrigen.lock.unlock();
                }
            }
        }

        List<String> obtenerPaginasInterioresNoVacias() {
            List<String> result = new ArrayList<>();
            for (String p : this.paginas.subList(1, this.paginas.size() - 1)) {
                if (!p.isEmpty()) result.add(p);
            }
            return result;
        }
    }

    public static void main(String[] args) {
        for (int i = 0; i < 100; i++) {
            final Libro libroA = new Libro(List.of("Portada-A","A1","A2","","A3","A4","Contraportada-A"));
            final Libro libroB = new Libro(List.of("Portada-B","B1","B2","B3","","B4","Contraportada-B"));

            try (var executor = Executors.newVirtualThreadPerTaskExecutor()) {
                executor.submit(() -> libroA.copiarPaginas(libroB));
                executor.submit(() -> libroB.copiarPaginas(libroA));
            }

            System.out.println("Libro A: " + libroA.paginas);
            System.out.println("Libro B: " + libroB.paginas);
            System.out.println();
        }
    }
}

🌺 Database2000

Una Database escribe 2000 veces una letra para formar una línea. Cuatro Writer lo hacen en bucle durante 1 segundo. Sin sincronización, las letras se mezclan entre sí.

Objetivo: cada línea escrita en data debe contener exactamente 2000 repeticiones de la misma letra, sin mezclas.

Código de partida

import java.util.concurrent.Executors;
import java.util.concurrent.TimeUnit;

public class Main {
    static class Database {
        String data = "";

        void write(String value) {
            for (int i = 0; i < 2000; i++) {
                this.data += value;
            }
            this.data += "\n";
        }
    }

    static class Writer implements Runnable {
        int numberOfWrites = 0;
        final Database database;
        final String letra;

        Writer(Database database, String letra) {
            this.database = database;
            this.letra = letra;
        }

        @Override
        public void run() {
            while (true) {
                database.write(letra);
                numberOfWrites++;
            }
        }
    }

    public static void main(String[] args) throws InterruptedException {
        var database = new Database();
        var writer1 = new Writer(database, "a");
        var writer2 = new Writer(database, "b");
        var writer3 = new Writer(database, "c");
        var writer4 = new Writer(database, "d");

        var executor = Executors.newVirtualThreadPerTaskExecutor();
        executor.submit(writer1);
        executor.submit(writer2);
        executor.submit(writer3);
        executor.submit(writer4);

        executor.awaitTermination(1, TimeUnit.SECONDS);

        System.out.println(database.data);
        System.out.println(writer1.numberOfWrites);
        System.out.println(writer2.numberOfWrites);
        System.out.println(writer3.numberOfWrites);
        System.out.println(writer4.numberOfWrites);
    }
}

Pista de diseño

  • 1
    Añade un Lock lock = new ReentrantLock(); a Database.
  • 2
    En write(), adquiere el lock antes del bucle y libéralo en finally.
  • 3
    Necesitarás que el bucle while(true) de Writer.run() se pueda interrumpir; comprueba Thread.currentThread().isInterrupted().
import java.util.concurrent.Executors;
import java.util.concurrent.TimeUnit;
import java.util.concurrent.locks.Lock;
import java.util.concurrent.locks.ReentrantLock;

public class Main {
    static class Database {
        String data = "";
        final Lock lock = new ReentrantLock();

        void write(String value) {
            lock.lock();
            try {
                for (int i = 0; i < 2000; i++) {
                    this.data += value;
                }
                this.data += "\n";
            } finally {
                lock.unlock();
            }
        }
    }

    static class Writer implements Runnable {
        int numberOfWrites = 0;
        final Database database;
        final String letra;

        Writer(Database database, String letra) {
            this.database = database;
            this.letra = letra;
        }

        @Override
        public void run() {
            while (!Thread.currentThread().isInterrupted()) {
                database.write(letra);
                numberOfWrites++;
            }
        }
    }

    public static void main(String[] args) throws InterruptedException {
        var database = new Database();
        var writer1 = new Writer(database, "a");
        var writer2 = new Writer(database, "b");
        var writer3 = new Writer(database, "c");
        var writer4 = new Writer(database, "d");

        var executor = Executors.newVirtualThreadPerTaskExecutor();
        executor.submit(writer1);
        executor.submit(writer2);
        executor.submit(writer3);
        executor.submit(writer4);

        executor.awaitTermination(1, TimeUnit.SECONDS);
        executor.shutdownNow();

        System.out.println(database.data.lines()
            .filter(l -> !l.isEmpty())
            .filter(l -> l.chars().distinct().count() > 1)
            .count() + " líneas mezcladas");   // debería ser 0

        System.out.println("Escrituras a: " + writer1.numberOfWrites);
        System.out.println("Escrituras b: " + writer2.numberOfWrites);
        System.out.println("Escrituras c: " + writer3.numberOfWrites);
        System.out.println("Escrituras d: " + writer4.numberOfWrites);
    }
}

⚛️ Variables Atómicas

El paquete java.util.concurrent.atomic proporciona clases que realizan operaciones indivisibles sobre valores sin necesidad de synchronized ni locks.

AtomicInteger AtomicLong AtomicReference compareAndSet getAndUpdate

Teoría

Teoría

¿Qué significa "atómico"?

Una operación atómica se ejecuta de forma indivisible: nunca se puede observar en un estado intermedio por otro thread. Una operación como i++ no es atómica; en realidad son tres pasos (leer, sumar, escribir) que pueden ser interrumpidos.

Ventaja: Las operaciones atómicas son generalmente más rápidas que synchronized porque usan instrucciones CAS (Compare-And-Swap) del procesador en lugar de bloquear el thread.

AtomicInteger — métodos principales

MétodoDescripciónValor devuelto
get()Lee el valor actualvalor actual
set(n)Establece el valorvoid
incrementAndGet()Suma 1 y devuelve el nuevo valorvalor después
getAndIncrement()Devuelve el valor actual y luego suma 1valor antes
decrementAndGet()Resta 1 y devuelve el nuevo valorvalor después
addAndGet(n)Suma n y devuelve el nuevo valorvalor después
getAndUpdate(fn)Aplica función y devuelve el valor anteriorvalor antes
updateAndGet(fn)Aplica función y devuelve el valor nuevovalor después
compareAndSet(exp, upd)Si el valor == exp, lo pone a updtrue si se actualizó
 
AtomicInteger contador = new AtomicInteger(0);

contador.incrementAndGet();   // → 1
contador.getAndIncrement();   // → devuelve 1, queda en 2
contador.addAndGet(10);       // → 12
contador.updateAndGet(v -> v * 2); // → 24
contador.compareAndSet(24, 0);     // → true, queda en 0

AtomicLong

Igual que AtomicInteger pero con tipo long. Ideal para contadores de gran volumen (IDs, timestamps, etc.).

AtomicLong idCounter = new AtomicLong(0);
long nuevoId = idCounter.incrementAndGet();  // genera IDs únicos de forma segura

AtomicReference<T>

Permite reemplazar objetos de forma atómica.

MétodoDescripción
get()Devuelve el objeto actual
set(v)Reemplaza el objeto
getAndSet(v)Devuelve el anterior y coloca el nuevo
getAndUpdate(fn)Aplica función; devuelve el valor anterior
updateAndGet(fn)Aplica función; devuelve el valor nuevo
compareAndSet(exp, upd)Actualiza solo si el valor actual == exp (por referencia)
 
record UserProfile(String name, int age) {}

AtomicReference<UserProfile> ref =
    new AtomicReference<>(new UserProfile("Ana", 25));

// Actualización incondicional
ref.set(new UserProfile("Pedro", 30));

// Actualización condicional por nombre
ref.getAndUpdate(p ->
    p.name().equals("Pedro") ? new UserProfile("Luis", 28) : p
);

// CAS — solo si el objeto es exactamente el esperado
UserProfile esperado = ref.get();
UserProfile nuevo    = new UserProfile("María", 22);
boolean actualizado  = ref.compareAndSet(esperado, nuevo);

Nota: compareAndSet compara por referencia (==), no por equals(). Es el mecanismo CAS de hardware.

Patrón CAS en bucle (Compare-And-Swap)

Cuando la actualización depende del valor actual y puede fallar si otro thread lo cambió mientras tanto, se usa un bucle hasta que la operación tenga éxito:

// Incrementar solo si el valor es menor que MAX
AtomicInteger counter = new AtomicInteger(0);
final int MAX = 5;

boolean success = false;
while (!success) {
    int current = counter.get();
    if (current >= MAX) {
        System.out.println("Límite alcanzado");
        break;
    }
    success = counter.compareAndSet(current, current + 1);
}

Esto es equivalente a lo que hacen internamente getAndUpdate y updateAndGet.

🐦 DeepPurpleSky — tres formas de sincronizar

Un Post tiene contadores de meGustas y rePublicaciones. Se envían 5.000.000 operaciones de cada tipo de forma concurrente. Los contadores dan valores incorrectos.

Implementa la solución con synchronized, con Lock y con atomic. Compara el tiempo de ejecución.

Código de partida

import java.time.LocalDateTime;
import java.time.temporal.ChronoUnit;
import java.util.concurrent.Executors;

public class Main {

    static class Post {
        int meGustas = 0;
        int rePublicaciones = 0;

        void meGusta() { meGustas++; }
        void rePublicar() { rePublicaciones++; }
    }

    public static void main(String[] args) {
        var startTime = LocalDateTime.now();
        Post post = new Post();

        try (var executor = Executors.newVirtualThreadPerTaskExecutor()) {
            for (int i = 0; i < 5_000_000; i++) {
                executor.submit(post::meGusta);
                executor.submit(post::rePublicar);
            }
        }

        System.out.println(post.meGustas);       // debería ser 5000000
        System.out.println(post.rePublicaciones); // debería ser 5000000
        System.out.println("Millis: " + ChronoUnit.MILLIS.between(startTime, LocalDateTime.now()));
    }
}

static class Post {

    int rePublicaciones = 0;

import java.util.concurrent.locks.*;

static class Post {
    int meGustas = 0;
    int rePublicaciones = 0;
    final Lock lock = new ReentrantLock();

    void meGusta() {
        lock.lock();
        try { meGustas++; }
        finally { lock.unlock(); }
    }

    void rePublicar() {
        lock.lock();
        try { rePublicaciones++; }
        finally { lock.unlock(); }
    }
}

import java.util.concurrent.atomic.*;

static class Post {
    AtomicInteger meGustas       = new AtomicInteger(0);
    AtomicInteger rePublicaciones = new AtomicInteger(0);

    void meGusta()    { meGustas.incrementAndGet(); }
    void rePublicar() { rePublicaciones.incrementAndGet(); }
}

👨‍👨‍👦‍👦 Control de acceso — máximo de usuarios

Sistema que limita el número máximo de usuarios simultáneos (por ejemplo, 5).

  • 1
    connect(): incrementa el contador si hay hueco disponible; si no, muestra "acceso bloqueado".
  • 2
    disconnect(): decrementa, asegurando que no quede negativo.
  • 3
    Nunca se debe superar el límite máximo aunque muchos threads conecten a la vez.

Clave: usa un bucle con compareAndSet para que la comprobación y el incremento sean atómicos juntos.

import java.util.concurrent.Executors;
import java.util.concurrent.atomic.AtomicInteger;

public class Main {

    static class AccessControl {
        private final AtomicInteger currentUsers = new AtomicInteger(0);
        private final int maxUsers;

        AccessControl(int maxUsers) { this.maxUsers = maxUsers; }

        boolean connect() {
            while (true) {
                int current = currentUsers.get();
                if (current >= maxUsers) {
                    System.out.println("Acceso bloqueado. Usuarios: " + current + "/" + maxUsers);
                    return false;
                }
                if (currentUsers.compareAndSet(current, current + 1)) {
                    System.out.println("Conectado. Usuarios: " + (current + 1) + "/" + maxUsers);
                    return true;
                }
                // otro thread lo cambió mientras tanto, reintentar
            }
        }

        void disconnect() {
            while (true) {
                int current = currentUsers.get();
                if (current <= 0) return;
                if (currentUsers.compareAndSet(current, current - 1)) {
                    System.out.println("Desconectado. Usuarios: " + (current - 1) + "/" + maxUsers);
                    return;
                }
            }
        }
    }

    public static void main(String[] args) throws InterruptedException {
        var control = new AccessControl(5);

        try (var executor = Executors.newVirtualThreadPerTaskExecutor()) {
            for (int i = 0; i < 20; i++) {
                executor.submit(() -> {
                    boolean connected = control.connect();
                    if (connected) {
                        try { Thread.sleep(100); }
                        catch (InterruptedException e) { Thread.currentThread().interrupt(); }
                        control.disconnect();
                    }
                });
            }
        }
    }
}

🗣 Actualizar perfil — AtomicReference

Gestiona un UserProfile compartido con tres métodos de actualización:

  • updateProfile(name, age) → usa set() — actualización incondicional.
  • updateProfile(expectedName, newName, age) → usa getAndUpdate() — solo actualiza si el nombre coincide.
  • updateProfile(expected, newProfile) → usa compareAndSet() — solo si la referencia es la misma.
import java.util.concurrent.Executors;
import java.util.concurrent.atomic.AtomicReference;

public class Main {

    record UserProfile(String name, int age) {}

    static class UserProfileUpdater {
        private final AtomicReference<UserProfile> ref =
            new AtomicReference<>(new UserProfile("Juan Doe", 25));

        // 1. Actualización incondicional
        void updateProfile(String newName, int newAge) {
            ref.set(new UserProfile(newName, newAge));
            System.out.println("set() → " + ref.get());
        }

        // 2. Solo si el nombre actual coincide
        void updateProfile(String expectedName, String newName, int newAge) {
            UserProfile prev = ref.getAndUpdate(p ->
                p.name().equals(expectedName) ? new UserProfile(newName, newAge) : p
            );
            if (prev.name().equals(expectedName))
                System.out.println("getAndUpdate() actualizó: " + ref.get());
            else
                System.out.println("getAndUpdate() no actualizó (nombre='"+prev.name()+"' ≠ '"+expectedName+"')");
        }

        // 3. Solo si la referencia es exactamente la esperada
        boolean updateProfile(UserProfile expected, UserProfile newProfile) {
            boolean ok = ref.compareAndSet(expected, newProfile);
            System.out.println("compareAndSet() " + (ok ? "éxito" : "falló") + " → " + ref.get());
            return ok;
        }

        UserProfile getProfile() { return ref.get(); }
    }

    public static void main(String[] args) {
        var updater = new UserProfileUpdater();

        try (var executor = Executors.newVirtualThreadPerTaskExecutor()) {
            executor.submit(() -> updater.updateProfile("Nuevo1", 30));
            executor.submit(() -> updater.updateProfile("Juan Doe", "Pedro", 28));
            UserProfile expected = updater.getProfile();
            executor.submit(() -> updater.updateProfile(expected, new UserProfile("María", 22)));
        }

        System.out.println("Perfil final: " + updater.getProfile());
    }
}

🧎🏽‍♀️ TaskState — máquina de estados atómica

Implementa la gestión del estado de una tarea con transiciones válidas: INIT → PROCESSING → COMPLETED.

  • 1
    Clase Task con AtomicReference<TaskState> (enum: INIT, PROCESSING, COMPLETED).
  • 2
    setState(newState): valida la transición con compareAndSet.
  • 3
    run(): cambia a PROCESSING, llama a perform(), cambia a COMPLETED.
  • 4
    perform(): método abstracto. Varios threads intentan ejecutar la misma tarea — solo uno lo consigue.
import java.util.concurrent.Executors;
import java.util.concurrent.atomic.AtomicReference;

public class Main {

    enum TaskState { INIT, PROCESSING, COMPLETED }

    abstract static class Task {
        private final AtomicReference<TaskState> state =
            new AtomicReference<>(TaskState.INIT);

        TaskState getState() { return state.get(); }

        boolean setState(TaskState newState) {
            TaskState current = state.get();
            boolean valid = switch (newState) {
                case PROCESSING -> current == TaskState.INIT;
                case COMPLETED  -> current == TaskState.PROCESSING;
                default         -> false;
            };
            if (!valid) return false;
            return state.compareAndSet(current, newState);
        }

        void run() {
            if (!setState(TaskState.PROCESSING)) {
                System.out.println(Thread.currentThread().getName()
                    + " no pudo iniciar la tarea (estado=" + getState() + ")");
                return;
            }
            System.out.println(Thread.currentThread().getName() + " ejecutando tarea...");
            perform();
            setState(TaskState.COMPLETED);
            System.out.println(Thread.currentThread().getName() + " tarea completada.");
        }

        abstract void perform();
    }

    public static void main(String[] args) {
        Task tarea = new Task() {
            @Override
            public void perform() {
                try { Thread.sleep(200); } catch (InterruptedException e) { Thread.currentThread().interrupt(); }
                System.out.println("  [perform] trabajo hecho por " + Thread.currentThread().getName());
            }
        };

        try (var executor = Executors.newVirtualThreadPerTaskExecutor()) {
            for (int i = 0; i < 5; i++) {
                executor.submit(tarea::run);
            }
        }

        System.out.println("Estado final: " + tarea.getState());
    }
}

🏮 Colecciones Concurrentes

El paquete java.util.concurrent ofrece implementaciones thread-safe del Java Collections Framework. Definen una relación happens-before entre escrituras y lecturas.

ConcurrentHashMap CopyOnWriteArrayList BlockingQueue ConcurrentSkipListMap

Teoría

Teoría

¿Por qué no usar HashMap o ArrayList en threads?

Las colecciones estándar (HashMap, ArrayList, etc.) no son thread-safe. Si varios threads las modifican simultáneamente puede producirse:

  • Race condition: dos threads leen el mismo valor, lo modifican, y uno sobreescribe el trabajo del otro.
  • ConcurrentModificationException: un thread modifica la colección mientras otro la itera.
  • Datos corruptos: estructuras internas incoherentes (nodos perdidos en HashMap, etc.).

ConcurrentHashMap

Versión thread-safe de HashMap. No bloquea toda la tabla; divide el mapa en segmentos y bloquea solo el necesario.

MétodoDescripción
put(k,v)Inserta/reemplaza de forma segura
putIfAbsent(k,v)Solo inserta si la clave no existe. Atómico.
computeIfAbsent(k, fn)Si la clave no existe, calcula el valor con fn y lo inserta. Atómico.
compute(k, fn)Aplica fn(clave, valorActual→nuevoValor). Atómico. Si resulta null, elimina.
merge(k, v, fn)Si no existe: inserta v. Si existe: aplica fn(valorAnterior, v). Atómico.
getOrDefault(k, def)Devuelve el valor o el defecto si no existe
 
ConcurrentHashMap<String, Integer> conteo = new ConcurrentHashMap<>();

// Contar palabras de forma segura:
conteo.merge("manzana", 1, Integer::sum);

// Crear lista si la clave no existe:
ConcurrentHashMap<String, List<String>> grupos = new ConcurrentHashMap<>();
grupos.computeIfAbsent("admin", k -> new ArrayList<>()).add("user1");

// Actualizar solo si existe:
conteo.compute("manzana", (k, v) -> v == null ? 1 : v + 1);

CopyOnWriteArrayList

Versión thread-safe de ArrayList. Cada modificación (add, set, remove) crea una copia nueva del array subyacente. Los iteradores trabajan sobre la copia anterior, por lo que nunca lanzan ConcurrentModificationException.

Cuándo usarla: listas donde las lecturas son mucho más frecuentes que las escrituras. Si hay muchas escrituras, el coste de copiar el array en cada operación es elevado.

CopyOnWriteArrayList<String> lista = new CopyOnWriteArrayList<>();

// Múltiples threads pueden añadir de forma segura:
lista.add("elemento");

// Iteración segura aunque otros threads modifiquen la lista:
for (String item : lista) {
    System.out.println(item);
}

Patrón seguro: ConcurrentHashMap + CopyOnWriteArrayList

Combinación muy común para estructuras de tipo "mapa de listas":

// Mapa de grupos → lista de usuarios
ConcurrentHashMap<String, CopyOnWriteArrayList<String>> grupos =
    new ConcurrentHashMap<>();

// Añadir usuario a grupo (creando el grupo si no existe)
grupos.computeIfAbsent("grupo1", k -> new CopyOnWriteArrayList<>())
      .add("usuario1");

// Eliminar usuario de grupo; si queda vacío, eliminar grupo
grupos.computeIfPresent("grupo1", (k, v) -> {
    v.remove("usuario1");
    return v.isEmpty() ? null : v;  // null → elimina la entrada
});

🏨 Reservas de hotel

Sistema de reservas concurrente para habitaciones. Las reservas no deben solaparse en la misma habitación.

  • 1
    Usa un ConcurrentHashMap<String, List<Reserva>> (habitación → lista de reservas).
  • 2
    Al reservar, comprueba que no hay solapamiento con reservas existentes.
  • 3
    La comprobación + inserción debe ser atómica para la misma habitación (usa synchronized sobre la lista de esa habitación).
import java.time.LocalTime;
import java.util.ArrayList;
import java.util.List;
import java.util.concurrent.ConcurrentHashMap;
import java.util.concurrent.Executors;

public class Main {

    record Reserva(String usuario, String habitacion, LocalTime inicio, LocalTime fin) {}

    static class SistemaReservas {
        private final ConcurrentHashMap<String, List<Reserva>> reservas =
            new ConcurrentHashMap<>();

        boolean reservar(String usuario, String habitacion, LocalTime inicio, LocalTime fin) {
            // Obtener o crear la lista de reservas para esta habitación
            List<Reserva> listaHab = reservas.computeIfAbsent(habitacion, k -> new ArrayList<>());

            // Sincronizar sobre la lista específica de la habitación
            synchronized (listaHab) {
                boolean solapamiento = listaHab.stream().anyMatch(r ->
                    inicio.isBefore(r.fin()) && fin.isAfter(r.inicio())
                );
                if (solapamiento) {
                    System.out.println("❌ " + usuario + ": " + habitacion +
                        " " + inicio + "-" + fin + " ya está ocupada");
                    return false;
                }
                listaHab.add(new Reserva(usuario, habitacion, inicio, fin));
                System.out.println("✅ " + usuario + ": " + habitacion +
                    " " + inicio + "-" + fin + " reservada");
                return true;
            }
        }
    }

    public static void main(String[] args) {
        var sistema = new SistemaReservas();

        try (var executor = Executors.newVirtualThreadPerTaskExecutor()) {
            executor.submit(() -> sistema.reservar("Ana",   "101", LocalTime.of(9,0),  LocalTime.of(11,0)));
            executor.submit(() -> sistema.reservar("Pedro", "101", LocalTime.of(10,0), LocalTime.of(12,0))); // solapamiento
            executor.submit(() -> sistema.reservar("Luis",  "101", LocalTime.of(11,0), LocalTime.of(13,0)));
            executor.submit(() -> sistema.reservar("María", "102", LocalTime.of(9,0),  LocalTime.of(11,0)));
            executor.submit(() -> sistema.reservar("Juan",  "102", LocalTime.of(9,0),  LocalTime.of(10,0))); // solapamiento
        }
    }
}

🎎 Grupos de chat

Sistema para gestionar grupos de chat de forma concurrente. Permite añadir/eliminar grupos y usuarios dentro de ellos.

  • addUserToGroup(user, group): añade usuario; crea el grupo si no existe. Devuelve false si ya estaba.
  • removeUserFromGroup(user, group): quita usuario; elimina el grupo si queda vacío. Devuelve false si no estaba.
  • deleteGroup(group): elimina el grupo. Devuelve false si no existía.
import java.util.List;
import java.util.Random;
import java.util.Set;
import java.util.concurrent.ConcurrentHashMap;
import java.util.concurrent.CopyOnWriteArraySet;
import java.util.concurrent.Executors;
import java.util.concurrent.TimeUnit;

public class Main {

    static class GroupsManager {
        private final ConcurrentHashMap<String, Set<String>> grupos =
            new ConcurrentHashMap<>();

        boolean addUserToGroup(String user, String group) {
            Set<String> members = grupos.computeIfAbsent(group,
                k -> new CopyOnWriteArraySet<>());
            boolean added = members.add(user);
            if (added) System.out.println("+ " + user + " → " + group);
            return added;
        }

        boolean removeUserFromGroup(String user, String group) {
            Set<String> members = grupos.get(group);
            if (members == null) return false;
            boolean removed = members.remove(user);
            if (removed) {
                grupos.computeIfPresent(group, (k, v) -> v.isEmpty() ? null : v);
                System.out.println("- " + user + " ← " + group);
            }
            return removed;
        }

        boolean deleteGroup(String group) {
            boolean existed = grupos.remove(group) != null;
            if (existed) System.out.println("🗑 " + group + " eliminado");
            return existed;
        }

        void printState() {
            grupos.forEach((g, users) ->
                System.out.println("  [" + g + "] " + users));
        }
    }

    public static void main(String[] args) throws InterruptedException {
        var manager = new GroupsManager();
        var users  = List.of("usera","userb","userc","userd","usere","userf","userg","userh");
        var groups = List.of("group1","group2","group3","group4");
        var rnd = new Random();

        var executor = Executors.newVirtualThreadPerTaskExecutor();

        for (int i = 0; i < 30; i++) {
            final String user  = users.get(rnd.nextInt(users.size()));
            final String group = groups.get(rnd.nextInt(groups.size()));
            if (rnd.nextBoolean()) {
                executor.submit(() -> manager.addUserToGroup(user, group));
            } else {
                executor.submit(() -> manager.removeUserFromGroup(user, group));
            }
        }

        executor.awaitTermination(2, TimeUnit.SECONDS);
        executor.shutdown();

        System.out.println("\nEstado final:");
        manager.printState();
    }
}

🫂 Messenger — mensajes con expiración

Sistema de mensajería donde múltiples threads de usuario envían mensajes y un thread de servicio elimina los mensajes con más de 3 segundos de antigüedad.

  • 1
    Usa CopyOnWriteArrayList<Mensaje> para la lista compartida.
  • 2
    Threads de usuario: añaden mensajes aleatorios cada 2-4 segundos.
  • 3
    Thread de servicio: cada 100ms elimina mensajes con más de 3 segundos.
  • 4
    El número de mensajes debería mantenerse cercano al número de usuarios.
import java.time.LocalDateTime;
import java.time.temporal.ChronoUnit;
import java.util.List;
import java.util.Random;
import java.util.concurrent.CopyOnWriteArrayList;
import java.util.concurrent.Executors;
import java.util.concurrent.TimeUnit;

public class Main {

    record Mensaje(String text, LocalDateTime date) {
        Mensaje(String text) { this(text, LocalDateTime.now()); }
    }

    public static void main(String[] args) throws InterruptedException {
        var mensajes = new CopyOnWriteArrayList<Mensaje>();
        var rnd = new Random();
        var usuarios = List.of("Ana","Pedro","Luis","María","Juan");

        var executor = Executors.newVirtualThreadPerTaskExecutor();

        // Threads de usuario
        for (String usuario : usuarios) {
            executor.submit(() -> {
                while (!Thread.currentThread().isInterrupted()) {
                    try {
                        String texto = usuario + ": msg-" + rnd.nextInt(1000);
                        mensajes.add(new Mensaje(texto));
                        System.out.println("[+" + usuario + "] total mensajes: " + mensajes.size());
                        Thread.sleep(2000 + rnd.nextInt(2000)); // 2-4 segundos
                    } catch (InterruptedException e) {
                        Thread.currentThread().interrupt();
                    }
                }
            });
        }

        // Thread de servicio — limpia mensajes > 3 segundos
        executor.submit(() -> {
            while (!Thread.currentThread().isInterrupted()) {
                try {
                    LocalDateTime limite = LocalDateTime.now().minus(3, ChronoUnit.SECONDS);
                    mensajes.removeIf(m -> m.date().isBefore(limite));
                    Thread.sleep(100);
                } catch (InterruptedException e) {
                    Thread.currentThread().interrupt();
                }
            }
        });

        executor.awaitTermination(15, TimeUnit.SECONDS);
        executor.shutdownNow();

        System.out.println("\nMensajes finales: " + mensajes.size());
    }
}

🤼‍♀️ Lista de la compra colaborativa

Cuatro threads operan concurrentemente sobre una lista de la compra: uno añade items, otro cambia su estado, otro elimina items y otro muestra la lista.

Usa el siguiente record para los items:

record Item(String description, AtomicReference<State> state) {
    enum State { CHECKED, UNCHECKED }

    Item(String description) {
        this(description, new AtomicReference<>(Item.State.UNCHECKED));
    }

    void setState(State newState) { state.set(newState); }
}

¿Por qué AtomicReference en el record? Los records son inmutables: no puedes cambiar sus campos. Pero AtomicReference permite mutar el estado que apunta, manteniendo la referencia constante.

import java.util.Random;
import java.util.UUID;
import java.util.concurrent.CopyOnWriteArrayList;
import java.util.concurrent.Executors;
import java.util.concurrent.TimeUnit;
import java.util.concurrent.atomic.AtomicReference;

public class Main {

    record Item(String description, AtomicReference<Item.State> state) {
        enum State { CHECKED, UNCHECKED }

        Item(String description) {
            this(description, new AtomicReference<>(Item.State.UNCHECKED));
        }

        void setState(State newState) { state.set(newState); }

        @Override public String toString() {
            return "[" + (state.get() == State.CHECKED ? "✓" : " ") + "] " + description;
        }
    }

    public static void main(String[] args) throws InterruptedException {
        var lista = new CopyOnWriteArrayList<Item>();
        var rnd   = new Random();

        var executor = Executors.newVirtualThreadPerTaskExecutor();

        // Thread 1: añadir items (máx 10)
        executor.submit(() -> {
            while (!Thread.currentThread().isInterrupted()) {
                try {
                    if (lista.size() < 10) {
                        lista.add(new Item(UUID.randomUUID().toString().substring(0, 6)));
                    }
                    Thread.sleep(300);
                } catch (InterruptedException e) { Thread.currentThread().interrupt(); }
            }
        });

        // Thread 2: cambiar estado de un item aleatorio
        executor.submit(() -> {
            while (!Thread.currentThread().isInterrupted()) {
                try {
                    if (!lista.isEmpty()) {
                        Item item = lista.get(rnd.nextInt(lista.size()));
                        item.setState(rnd.nextBoolean() ? Item.State.CHECKED : Item.State.UNCHECKED);
                    }
                    Thread.sleep(400);
                } catch (InterruptedException e) { Thread.currentThread().interrupt(); }
            }
        });

        // Thread 3: eliminar un item aleatorio
        executor.submit(() -> {
            while (!Thread.currentThread().isInterrupted()) {
                try {
                    if (!lista.isEmpty()) {
                        lista.remove(rnd.nextInt(lista.size()));
                    }
                    Thread.sleep(600);
                } catch (InterruptedException e) { Thread.currentThread().interrupt(); }
            }
        });

        // Thread 4: mostrar la lista
        executor.submit(() -> {
            while (!Thread.currentThread().isInterrupted()) {
                try {
                    System.out.println("--- Lista (" + lista.size() + " items) ---");
                    lista.forEach(System.out::println);
                    Thread.sleep(1500);
                } catch (InterruptedException e) { Thread.currentThread().interrupt(); }
            }
        });

        executor.awaitTermination(10, TimeUnit.SECONDS);
        executor.shutdownNow();
    }
}

🎁 Inventario de productos

Inventario con IDs autoincrement. Threads de compra reducen el stock; threads de reposición lo aumentan. Un thread muestra el inventario periódicamente.

  • 1
    Usa record Producto con int id, String nombre, AtomicInteger stock.
  • 2
    El id se genera con un static AtomicInteger en el record.
  • 3
    Inicializa 5 productos en un CopyOnWriteArrayList.
  • 4
    Varios threads simulan compras y reposiciones aleatorias.
import java.util.List;
import java.util.Random;
import java.util.concurrent.CopyOnWriteArrayList;
import java.util.concurrent.Executors;
import java.util.concurrent.TimeUnit;
import java.util.concurrent.atomic.AtomicInteger;

public class Main {

    record Producto(int id, String nombre, AtomicInteger stock) {
        private static final AtomicInteger nextId = new AtomicInteger(0);

        Producto(String nombre, int stockInicial) {
            this(nextId.incrementAndGet(), nombre, new AtomicInteger(stockInicial));
        }

        void aumentarStock(int cantidad) { stock.addAndGet(cantidad); }

        void disminuirStock(int cantidad) {
            stock.updateAndGet(s -> Math.max(0, s - cantidad));
        }

        @Override public String toString() {
            return String.format("Prod#%d %-12s stock: %d", id, nombre, stock.get());
        }
    }

    public static void main(String[] args) throws InterruptedException {
        var inventario = new CopyOnWriteArrayList<>(List.of(
            new Producto("Manzanas",  50),
            new Producto("Peras",     30),
            new Producto("Naranjas",  80),
            new Producto("Uvas",      20),
            new Producto("Mangos",    15)
        ));
        var rnd = new Random();

        var executor = Executors.newVirtualThreadPerTaskExecutor();

        // Threads de compra
        for (int i = 0; i < 3; i++) {
            executor.submit(() -> {
                while (!Thread.currentThread().isInterrupted()) {
                    try {
                        Producto p = inventario.get(rnd.nextInt(inventario.size()));
                        int cantidad = rnd.nextInt(5) + 1;
                        p.disminuirStock(cantidad);
                        Thread.sleep(100);
                    } catch (InterruptedException e) { Thread.currentThread().interrupt(); }
                }
            });
        }

        // Threads de reposición
        for (int i = 0; i < 2; i++) {
            executor.submit(() -> {
                while (!Thread.currentThread().isInterrupted()) {
                    try {
                        Producto p = inventario.get(rnd.nextInt(inventario.size()));
                        int cantidad = rnd.nextInt(10) + 5;
                        p.aumentarStock(cantidad);
                        Thread.sleep(200);
                    } catch (InterruptedException e) { Thread.currentThread().interrupt(); }
                }
            });
        }

        // Thread de visualización
        executor.submit(() -> {
            while (!Thread.currentThread().isInterrupted()) {
                try {
                    System.out.println("\n=== Inventario ===");
                    inventario.forEach(System.out::println);
                    Thread.sleep(2000);
                } catch (InterruptedException e) { Thread.currentThread().interrupt(); }
            }
        });

        executor.awaitTermination(10, TimeUnit.SECONDS);
        executor.shutdownNow();
    }
}
